# Implementation Context

This document is the current handoff note for future chats working on `ai-course-summarizer`.

## Project Summary

- Angular 21 standalone application.
- Primary goal: turn raw lecture transcripts into structured session JSON and render a dashboard for each course session.
- Current syllabus in code contains 19 sessions.
- Static persisted session data lives in `public/data/session-<id>.json`.
- The app currently supports live generation from the browser through the Gemini Developer API.

## Core Files

- `src/app/core/gemini.ts`: Gemini SDK integration and structured JSON generation.
- `src/app/core/syllabus.service.ts`: source of truth for session metadata, phase grouping, and progress.
- `src/app/session/session.ts`: session page state, loading, live generation, and player interactions.
- `src/app/session/session-helpers.ts`: pure helper logic for ratings, filtering, timestamps, and runtime estimation.
- `src/app/app.ts`: shell component, sidebar, theme, and route-level structure.
- `src/environments/environment.ts`: local-only environment file for Gemini API key.
- `src/environments/environment.sample.ts`: safe template for local setup.

## Generation Pipeline

The generation pipeline is intentionally schema-driven.

1. The session page collects a YouTube URL and transcript in the admin HUD.
2. While generating, the form is replaced with an animated processing state UI.
3. `GeminiService.processTranscript()` sends the full transcript to `@google/genai` with `temperature: 0`.
4. The request uses:
   - `config.temperature: 0`
   - `config.systemInstruction`
   - `config.responseMimeType = 'application/json'`
   - `config.responseJsonSchema` for `GeneratedData`
5. The parsed JSON is validated at a light structural level before use.
6. UI-only question ratings are added after generation by `decorateGeneratedDataWithRatings()`.
7. Downloaded JSON merges in `videoUrl` from the admin URL field, then strips UI-only ratings before saving.
8. On next page load, `videoUrl` from the JSON is used to wire up the embedded player and auto-unlock the sidebar.

## Expected JSON Shape

The generated payload must match `GeneratedData`. The persisted file on disk also includes `videoUrl`, which is injected at download time â€” not generated by the model.

- `sessionId: number`
- `videoUrl?: string` â€” YouTube URL, admin-supplied, stored in JSON, never emitted by Gemini
- `sessionOverview: string`
- `instructorTakeaways: { title: string; body: string }[]`
  - expected count: exactly 6
- `summary: { timestamp: string; title: string }[]`
  - expected count: 10 to 16
- `qa: { timestamp?: string; speaker?: string; question: string; answer: string }[]`

Important:

- The model must not emit `rating` or `videoUrl`; both are added by the app after generation.
- Strings should be plain text only.
- Timestamps should remain in transcript order and use `MM:SS` or `HH:MM:SS` where possible.

## Current Gemini SDK Notes

- Package in use: `@google/genai`.
- The request shape:
  - `model`
  - `contents`
  - `config.temperature: 0` â€” deterministic output for reproducible extraction
  - `config.systemInstruction`
  - `config.responseMimeType = 'application/json'`
  - `config.responseJsonSchema` for `GeneratedData`
- The old top-level `systemInstruction` usage is no longer the intended pattern.
- No transcript slicing â€” the full transcript is sent. `gemini-2.5-flash` has a 1M token context window.
- The app does not use fallback string slicing for JSON.

## Recommended Gemini Model

For this app, the best default is:

- `gemini-2.5-flash`

Reasoning:

- It is stable, not deprecated, and not preview-only.
- Google describes it as the best price-performance model for low-latency tasks that require reasoning.
- It is materially cheaper than `gemini-2.5-pro` while being more reliable than `gemini-2.5-flash-lite` for structured transcript extraction.
- It supports a large context window, which matters for long transcripts.

Use these rules when selecting a model:

- Use `gemini-2.5-flash` for the default production path.
- Use `gemini-2.5-flash-lite` only when cost is the absolute top priority and occasional quality drops or retries are acceptable.
- Use `gemini-2.5-pro` only for especially difficult transcripts or when extraction quality clearly matters more than price.
- Avoid deprecated Gemini 2.0 models.
- Avoid preview models as the default unless there is a specific capability gap that stable models cannot satisfy.

## Pricing Guidance

Based on Google Gemini Developer API docs checked on 2026-04-06:

- `gemini-2.5-flash`
  - input: $0.30 / 1M tokens
  - output: $2.50 / 1M tokens
- `gemini-2.5-flash-lite`
  - input: $0.10 / 1M tokens
  - output: $0.40 / 1M tokens
- `gemini-2.5-pro`
  - input: $1.25 / 1M tokens for prompts <= 200k tokens
  - output: $10.00 / 1M tokens for prompts <= 200k tokens

Interpretation:

- `2.5-flash-lite` is the cheapest.
- `2.5-flash` is the best cost/reliability balance.
- `2.5-pro` is much more expensive and should not be the default here.

## Free Alternative

If cost or key exposure becomes a problem, the best free alternative is a local Ollama-based implementation.

- Ollama supports JSON and schema-shaped outputs.
- It avoids browser-exposed cloud API keys.
- It is best run behind a local Node proxy if the Angular app stays browser-only.
- Expect lower quality than Gemini on very long or noisy transcripts unless you add chunking.

## Security Notes

- `src/environments/environment.ts` is ignored by Git and is intended for local-only secrets.
- Do not commit a live Gemini API key.
- If a real key was committed at any point, rotate it in Google AI Studio and replace it locally.
- For browser use, prefer a restricted key with allowed referrer origins.
